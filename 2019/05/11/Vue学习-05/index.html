没有key时出现的事件问题

当执行 list 的 unshift() 方法(倒序插入)时，在添加元素时会出现：选定的checkbox之前添加了元素(元素被添加到了第一行)，并且选中的元素出现了 id 变化的情况，这不是我们期望看到的。为了避免这样现象，需要在迭代的位置，添加一个标志属性(:key=”item.id”)，以标志迭代出来的元素是唯一标志的，如下：

注意：v-for 循环的时候，key 属性只能使用 number 或者 String ；
key 在使用的时候的只能使用 v-bind 进行绑定
在组建中使用 v-for 循环的时候，若 v-for 有问题，必须在使用 v-bind 绑定唯一字符串或数字
